# Neo Anki 0.2.1

Neo Anki 0.2.1 is the first desktop preview to include the reviewed extension marketplace and the complete Workspace v4 durability fixes. It is intended for real desktop use with independent backups while the broader mobile and hosted-service release gates remain open.

## Highlights

- Adds the reviewed SDK 2 extension marketplace, with signed packages, capability review, isolated workers, sandboxed UI, and atomic extension installation and recovery.
- Makes desktop note edits durable across Anki migration, rapid follow-up edits, application restarts, and overlapping save boundaries.
- Preserves Anki note types, named fields, templates, CSS, deck ownership, scheduling data, flags, suspension, bury state, media, and review history through the documented Workspace v4 compatibility layer.
- Ships production dependency and TypeScript updates together with the Web Crypto and byte-boundary compatibility fixes required by the updated toolchain.
- Runs desktop acceptance journeys with hidden, recovery-aware Electron windows and stabilizes the TTS settings, cancellation, and persistence journey without retrying unchanged failures.
- Keeps release signing credentials optional: unsigned packages are still launch-tested and accompanied by checksums, CycloneDX SBOMs, GitHub provenance attestations, and platform-specific installation guidance.

## Compatibility and limits

- Neo Anki remains a pre-1.0 desktop preview and is not advertised as a complete drop-in Anki replacement. Supported import and export behavior is documented in `docs/anki-compatibility.md`.
- Keep an independent backup before importing an important collection. Migration uses a field-level preflight and retains verified checkpoints, but future Anki schemas may require compatibility updates.
- Mobile application-store binaries and hosted sync are not part of this release. Their physical-device and operational gates remain documented separately.
- macOS and Windows packages are unsigned. Follow the attached installation guide and verify the SHA-256 checksum and GitHub attestation before opening them.
- Automatic updates remain disabled. Install newer releases manually after verification.
