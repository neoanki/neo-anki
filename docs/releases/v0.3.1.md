# Neo Anki 0.3.1

Neo Anki 0.3.1 presents every marketplace extension as an independent end-user product. Names, descriptions, in-app labels, documentation, and marketplace copy now explain what each extension does instead of how it was separated from the application.

## Extensions

- Uses consistent names throughout Neo Anki: Card Timer, Collection Insights, Text to Speech, More Card Types, Image Occlusion, Anki & CSV Import/Export, Review Priorities, Goals & Saved Searches, and Learning Packs.
- Replaces architecture terms such as “isolated,” “bounded,” “extension-owned,” and “core-owned” in learner-facing copy with feature, privacy, and data-protection explanations.
- Makes onboarding, Plans & sharing, mobile audio labels, and extension downgrade warnings easier to understand.
- Keeps stable extension IDs, so existing installations upgrade without losing settings or extension data.

## SDK runtime

- Launches SDK 2 extension workers as ES modules, matching the package compiler output and allowing extensions to export code for their own unit tests.
- Keeps Electron E2E windows hidden when `NEO_ANKI_E2E_HEADLESS=1`, including extension repository integration tests.

## Verification

The release pipeline validates linting, type safety, unit coverage, browser behavior, hidden-window desktop integration, mobile export, extension packaging, licensing, platform installers, checksums, CycloneDX SBOMs, and build provenance.
