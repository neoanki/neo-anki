# Neo Anki 0.2.3

Neo Anki 0.2.3 moves Card Timer and Collection Insights out of the renderer bundle and into independently released, signed SDK 2 packages discovered through the extension marketplace. It also hardens reproducible extension verification across time zones and resolves a high-severity transitive build dependency advisory.

## Highlights

- Replaces the bundled Card Timer with the independently versioned [Card Timer 2.0.0](https://github.com/neoanki/neoanki-card-timer/releases/tag/v2.0.0) extension.
- Replaces the bundled Insights page with the independently versioned [Collection Insights 2.0.0](https://github.com/neoanki/neoanki-insights/releases/tag/v2.0.0) extension.
- Shows only signed, installable SDK 2 packages in the extension manager instead of presenting trusted core modules as extensions.
- Makes canonical extension archives byte-identical across host time zones, with regression coverage for UTC and Europe/Kyiv.
- Updates the transitive `shell-quote` dependency to a patched release and passes the high-severity dependency audit.

## Extension boundary

- Collection Insights 2.0.0 provides the current collection, deck, tag, health, and due summary available through the bounded `content:read` projection. Historical recall charts are not included because SDK 2 does not expose review history.
- More Card Types, Image Occlusion, Anki & CSV Import/Export, Review Priorities, Goals & Saved Searches, and Learning Packs remain trusted core features until invariant-preserving public contracts exist. They are no longer presented as installable extensions.
- The extraction boundary and next public SDK seams are recorded in `docs/audits/internal-extension-extraction-2026-07-21.md`.

## Compatibility and limits

- Neo Anki remains a pre-1.0 desktop preview and is not advertised as a complete drop-in Anki replacement. Supported import and export behavior is documented in `docs/anki-compatibility.md`.
- Keep an independent backup before importing an important collection. Migration retains the original package and a verified checkpoint, but future Anki schemas may require compatibility updates.
- Mobile application-store binaries and hosted sync are not part of this release. Their physical-device and operational gates remain documented separately.
- macOS and Windows packages are unsigned. Follow the attached installation guide and verify the SHA-256 checksum and GitHub attestation before opening them.
- Automatic updates remain disabled. Install newer releases manually after verification.
