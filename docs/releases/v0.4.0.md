# Neo Anki 0.4.0

Neo Anki 0.4.0 rebuilds the first-use journey around an empty, recoverable workspace and makes extensions a first-class destination. It is a desktop preview release; mobile-store binaries and hosted sync are not included.

## Clean first use

- Starts Fresh workspaces with zero knowledge items, practice prompts, reviews, assets, goals, or saved views.
- Lands on Today with direct actions to add knowledge, import from Anki, browse extensions, or explicitly load the sample workspace.
- Separates empty Library and Today states from filtered and caught-up states.
- Uses Prompt, Answer, knowledge item, practice prompt, and collection consistently.

## Data safety and review

- Blocks autosave after a recoverable load or migration failure and offers retry, preserved-source export, backup restore, and an explicit empty reset.
- Creates recovery checkpoints before destructive reset operations.
- Preserves the active review session when editing and displays explicit renderer failures instead of silently changing card meaning.

## Extensions and authoring

- Adds a full-screen Extensions hub with Browse, Installed, and Configure views.
- Adds live semantic appearance tokens, frame lifecycle handling, brokered file saving and external links, surface metadata, compatibility floors, and transactional activation rollback.
- Adds durable post-save authoring actions. Text to Speech can generate portable offline audio for a newly created knowledge item; prompt-type extensions provide structured validation and intended-media selection.
- Keeps Settings for Neo Anki preferences rather than marketplace and extension configuration.

## Platform notes

- macOS and Windows packages remain unsigned. Each artifact includes checksums, a CycloneDX SBOM, installation guidance, and GitHub build-provenance attestations.
- Browser and mobile clients can browse extension information but cannot install desktop extension workers.
- Automatic application updates remain disabled. Upgrade through a verified release download or the NeoAnki Homebrew tap.

## Verification

The release pipeline validates linting, type safety, unit coverage, browser journeys, hidden-window desktop integration, mobile export, extension packaging, licensing, platform installers, checksums, SBOMs, and build provenance before creating the draft release.
