# Neo Anki 0.2.2

Neo Anki 0.2.2 is a desktop durability patch that supersedes 0.2.1. It prevents an older rendered workspace snapshot from being persisted after a newer, explicitly awaited edit, including edits made immediately after an Anki migration.

## Fixed

- Makes the application data reference the synchronous authority for state transitions instead of allowing a delayed passive effect to move it backward.
- Discards persistence work from stale renders so an older snapshot cannot overwrite a completed note edit.
- Strengthens the imported-note regression to assert the final persisted named-field value, not merely that an intermediate change set contained it.
- Verifies the formerly failing hidden desktop import/edit/restart journey repeatedly before release.

## Compatibility and limits

- Neo Anki remains a pre-1.0 desktop preview and is not advertised as a complete drop-in Anki replacement. Supported import and export behavior is documented in `docs/anki-compatibility.md`.
- Keep an independent backup before importing an important collection. Migration retains the original package and a verified checkpoint, but future Anki schemas may require compatibility updates.
- Mobile application-store binaries and hosted sync are not part of this release. Their physical-device and operational gates remain documented separately.
- macOS and Windows packages are unsigned. Follow the attached installation guide and verify the SHA-256 checksum and GitHub attestation before opening them.
- Automatic updates remain disabled. Install newer releases manually after verification.
