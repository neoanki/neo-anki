# Neo Anki 0.2.0

Neo Anki 0.2.0 is a stabilization release focused on durable learning data, convergent sync, extension isolation, Anki migration boundaries, and consistent scheduling across desktop, browser, and native clients.

## Highlights

- Makes Workspace v4 the durable authority while preserving rich card variants, citations, media, occlusions, provenance, extension data, accessibility text, Trash tombstones and review history.
- Serializes desktop and mobile persistence so overlapping saves or sync cannot silently overwrite a newer review.
- Adds chunked, transactional encrypted sync with explicit restore operations, independent client-state fields, clock-skew limits, persistent service rate windows and bounded responses.
- Reissues extension capabilities safely across reload/disable/uninstall, cancels token-owned network work, bounds diagnostics, and removes the incomplete Browser Tab Sync snapshot transport.
- Applies per-preset scheduling and limits, removes buried siblings from active sessions, separates preview from learning evidence, improves typed-practice reveal, and aligns timing/metrics across clients.
- Stabilizes Anki collection identity, scopes collection replacement, rejects oversized imports before allocation, expands malformed/corpus coverage, and requires the pinned Anki oracle in release verification.
- Adds typed hash routes, global failed-save recovery, reduced-motion support, 44 px coarse-pointer targets, safe external-link handling, indexed projections/Insights, and lazy route/export loading.

## Compatibility and limits

- Neo Anki is not yet advertised as a drop-in Anki replacement. Exact supported import/export behavior is documented in `docs/anki-compatibility.md`.
- Imports over 64 MiB are refused before allocation and must be split. Entry, expansion, SQLite and Workspace envelope limits also apply.
- Browser sync protects data from the relay, but a same-origin script executing after unlock shares the browser key-use boundary. Use a signed desktop/native client for high-sensitivity collections.
- Mobile application-store binaries require the physical-device matrix in `docs/mobile-release-checklist.md`; this GitHub release contains desktop artifacts only.
- Automatic updates remain disabled. Install updates manually after verifying checksums and GitHub attestations.
