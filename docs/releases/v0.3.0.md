# Neo Anki 0.3.0

Neo Anki 0.3.0 completes the internal-extension extraction. Core now owns only learning-data, scheduling, review, persistence, recovery, sync, and extension trust/invariant authorities; every optional feature uses the public signed SDK 2 boundary.

## Included

- Removes More Card Types, Image Occlusion, Anki & CSV Import/Export, Review Priorities, Goals & Saved Searches, and Learning Packs from the renderer bundle and internal registry.
- Expands SDK 2.2.0 with manifest-declared prompt types, queue policies, Library presets, authoring/workspace/migration UI surfaces, and bounded DTOs.
- Adds a checkpointed migration broker: extensions may parse and preflight files, while core validates Workspace v4, creates rollback state, and commits atomically.
- Keeps shared-pack notes, cards, schema, provenance, and state in extension-owned atomic patches; concurrent field edits are preserved as explicit conflicts, and core scheduling/history is never overwritten by a pack update.
- Adds browser-safe compiler shims for dead SQLite compatibility branches without granting extension workers Node filesystem access.
- Publishes six standalone MIT-licensed repositories, signed reproducible releases, and marketplace listings with immutable hashes, publisher keys, exact permissions, and build provenance.

## Compatibility and limits

- The six extracted extensions require Neo Anki 0.3.0 and are not bundled. Install the capabilities you want from the marketplace.
- A basic forward prompt remains available with every core install. Missing or failing optional prompt renderers fall back to reviewable basic content.
- `content:migrate` is intentionally high authority and is shown during permission review. It has no network access in the first-party interoperability and shared-pack packages.
- Marketplace review and package signatures establish reviewed metadata, key possession, and artifact integrity; they are not warranties or real-world identity attestation.
- This release does not broaden Anki compatibility, durability, sync, mobile-readiness, or learning-effectiveness claims. Keep independent backups for important collections.
